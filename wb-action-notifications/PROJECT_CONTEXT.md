# Контекст проекта: /действия_уведомление

Этот подпроект формирует отдельное сообщение и Markdown-файл с действиями по товарам WB.

До согласования используется тестовый чат Пачки `39363429`.

## Команда

Ручной запуск через Пачку:

```text
/действия_уведомление
```

## Расписание

Плановое расписание через Cloudflare:

```text
5 5 * * *
```

Это `08:05 по МСК`.

Пока формат согласовывается, workflow по умолчанию отправляет в тестовый чат `39363429`. Для продового запуска нужно передать `pachca_chat_id` или настроить secret `PACHCA_CHAT_ID_ACTIONS`.

## Сегменты

- `НАСТРОИТЬ ЦЕНУ`
- `ВКЛЮЧИТЬ БЗО`
- `СОЗДАТЬ РК`
- `ПРОВЕРИТЬ АКТИВНОСТЬ РК`: только если FBO >= 50, РК есть, траты (3д) < 3 000 руб., оборачиваемость > 30 дней и категория не входит во временные исключения `CHECK_RK_EXCLUDED_CATEGORIES` (`Колготки` по умолчанию).
- `ВЫКЛЮЧИТЬ РК`

## Источники данных

- База Акинатора через `McpSql`: текущий FBO, карточки, цены, заказы, РК и траты РК.
- WB card API через FBO-модуль: БЗО, количество отзывов, рейтинг.
- Google Sheets поставок FBO: ближайшая поставка в горизонте 30 дней.
- Google Sheets ответственных: менеджер и маркетолог по артикулу WB.

## Источники ответственных

- Маркетолог: та же логика, что в FBO-уведомлениях, функция `load_marketer_by_article()` из `wb-fbo-supply-notifications/build_sheet_supplies_md.py`.
- Менеджер: та же логика, что в FBO-уведомлениях, функция `load_manager_by_article()` из `wb-fbo-supply-notifications/build_sheet_supplies_md.py`.

В сообщении:

- товары внутри сегмента выводятся одной строкой через символ `•`;
- сегмент `ПРОВЕРИТЬ АКТИВНОСТЬ РК` в сообщении группируется по маркетологу и категории: маркетолог -> категория -> количество SKU -> список артикулов;
- вложенные строки `↳` и разделители `-----` в этом отчете не используются;
- цена и БЗО призывают менеджера;
- БЗО дополнительно призывает Елену Ханжову через Pachca mention `<@user_id>`, который резолвится по API `/users` из запроса `PACHCA_BZO_MENTION_QUERY`;
- РК-действия призывают маркетолога;
- если ответственный не найден, выводится `не указан менеджер` или `не указан маркетолог`.
